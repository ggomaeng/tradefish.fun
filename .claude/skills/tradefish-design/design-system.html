tradeFish · design system · 2026-05-11

the tradeFish
design system.

Pixel-glitch candlestick fish on near-pure-black. Departure Mono everywhere, sharp pixel corners, glow instead of drop shadow, the 5-stop spectrum carrying every accent moment. Partner-built layout structure, components, and UX rigor — rebuilt in the terminal voice. One canvas for builders, askers, and the swarm.

FOUNDATIONS · 01

Brand

The mark is a candlestick fish — wicks for spine, body in the full magenta→mint spectrum. It carries the brand alone. The wordmark stays pixel-monospace because the mark is doing the heavy lifting.

The mark

Two sanctioned contexts: dark canvas (canonical) and the hero "bloom" — a soft spectrum halo behind the fish. Never re-render the mark in a single solid color. Never desaturate.

┌─ dark · canonical 120px+
┌─ hero · bloom 200px+ · once per surface

Wordmark

Departure Mono, lowercase product name tradeFish, tracking +0.04em. Always set in the foreground color — never apply the spectrum gradient to the wordmark itself. The mark beside the wordmark is optional once context is clear.

FOUNDATIONS · 02

Color

Near-pure-black canvas. The 5-stop spectrum carries every accent moment. Direction colors map to spectrum stops so charts and prediction cards speak the same vocabulary as the brand. No drop shadows — only glow.

Backgrounds & foreground

Four-step background ramp from canvas (--bg-0) to raised (--bg-3). Four-step foreground ramp from full text (--fg) to ghost (--fg-faintest). Use the lowest contrast that still reads — readability is the constraint, not maximum contrast.

FOUNDATIONS · 03

Spectrum

The logo gradient, sampled left-to-right. Every brand moment is built from these five stops. Use the full --grad-spectrum at most once per surface — wordmark, hero CTA, one chip. Single stops carry semantics: --spec-4 (cyan) = LIVE, --spec-5 (mint) = LONG, --spec-1 (magenta) = SHORT.

!
spectrum budget: the full --grad-spectrum appears at most once per surface. Single stops can repeat. HOLD sits OUT of the spectrum as bright grey — direction calls must dominate visually.
!
accent budget per row/card: exactly one hot signal carries the unit's purpose — the PnL, the live state, the call-to-action. Everything else in the same row recedes: avatars desaturate to ~55%, metadata wears .tf-tag (no border, no glow) instead of a status chip, ranks dim from #4 onwards. Saturated chip-cyan / chip-magenta / chip-mint are action states (LIVE, LONG, SHORT, HOLD) — never metadata.
!
chip vs tag — semantic split: .chip carries direction/status (LONG / SHORT / HOLD / LIVE) — colored border + bg + text, glow optional. .tf-tag carries visibility/scope/metadata (PUBLIC, VERIFIED, TIER · 01, DRAFT) — tinted bg only, no border, no glow. If "demoting" a chip makes the row breathe, it was always a tag.
FOUNDATIONS · 04

Typography

Three tiers, not one. Departure Mono is the "logo voice" — reserved for hero numerics and easter-egg moments. Geist Mono is the chrome — headings, labels, badges, button text. Geist sans is the body. The pixel face used to do all three jobs at once; everything read at the same loudness. The split gives the page a clear hierarchy without changing palette.

Tier 1 · Pixel — Departure Mono

Reserved. The pixel face only appears at hero scale or on a deliberate easter-egg. Never use it for chrome — it makes a button look as loud as a $1,284 PnL. --font-pixel.

Numerics

Departure Mono is monospaced by definition — every digit occupies the same advance width. Tabular numerics turned on globally. Never mix in a proportional sans for numbers; columns will jitter.

┌─ do · departure mono, tabular
+$1,284.30 +$ 406.10 +$ 12.05 −$ 178.40
┌─ don't · proportional sans
+$1,284.30 +$ 406.10 +$ 12.05 −$ 178.40
Stack: Departure Mono → JetBrains Mono → ui-monospace Always: font-variant-numeric: tabular-nums

FOUNDATIONS · 07

Glow (not shadow)

There are no drop shadows. Depth comes from three brand glows: --glow-cyan (live, primary), --glow-magenta (alert, risk), --glow-mint (settled, PnL+). Halo and bloom variants escalate the same hue.

PART · 02

Components

Neutral building blocks. Each one is direct, composable, never carries the brand alone. They recede so content can lead.

COMPONENT · 01

Six variants. Secondary .btn is the workhorse. .btn-primary is cyan-outline → cyan-fill on hover with a glow. .btn-sol is the single full-spectrum CTA per surface. .btn-danger & .btn-success trade in magenta/mint glow.

Every input + button has five canonical states: default, hover, focus (keyboard-only via :focus-visible), error (magenta border + glow), success (mint border), disabled (50% opacity + no-cursor). Mouse-clicks never trigger the focus ring — only Tab/arrow keys do.

COMPONENT · 11

Tags & field markers

Tiny visibility / scope markers. Distinct from .chip (which carries status colour) — tags label a field's nature: PUBLIC, PRIVATE, REQUIRED, DRAFT. 10px / 0.18em / sharp 2px corner. Borrowed from arena.colosseum.org, where every public field carries one.

COMPONENT · 12

Info panel

Blueprint-style informational callout. Four L-shaped corner ticks make it read as a technical drawing without spending much ink. Use for non-blocking guidance next to a form field — never for errors (use a magenta variant) or success (mint variant). Borrowed wholesale from Colosseum's "Please ensure we can access the repo" pattern.

PART · 05

Voice & tone

Lowercase product name tradeFish. Imperative, technical, sparse. Avoid SaaS hedging; never frame the swarm as winner-take-all.

┌─ do
"plug your agent into the swarm."
imperative · concrete · tells the user what to do next
✕ don't
"unlock AI-powered alpha generation today."
vague SaaS · promises returns · winner-take-all framing
┌─ do
"every answer becomes a paper trade."
honest about the MVP · names the mechanism
✕ don't
"guaranteed alpha · proven by AI."
guarantees nothing in markets · AI-as-magic
┌─ do
"the trade is over · the swarm got smarter."
cooperative · TradeWiki framing · settled outcome
✕ don't
"crush your competition on the leaderboard."
winner-take-all · against the cooperative thesis
